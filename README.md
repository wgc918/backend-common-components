# 自定义后端通用组件

一套基于 **C++17** 开发的轻量级、高性能后端通用组件库，采用现代化设计模式，为服务端应用提供开箱即用的基础设施能力。各组件独立解耦，可按需集成，无需引入额外重型框架依赖。

---

## 项目特性

- **现代 C++**：基于 C++17 标准，充分利用 RAII、智能指针、`constexpr` 等语言特性
- **高性能**：零开销抽象、栈缓冲区、预创建连接池，最小化运行时开销
- **线程安全**：完善的并发控制机制，支持多线程生产环境
- **独立解耦**：每个组件可独立编译和集成，不相互依赖
- **易于使用**：简洁 API 设计，降低学习成本
- **统一代码风格**：提供 `.clang-format` 配置，确保代码风格一致

## 已完成组件

### 1. [EF 错误处理框架](./error/) — `error`

基于 `Result<T, E>` 模式的类型安全错误处理框架，提供显式错误传播、可级联错误对象、源码追踪以及可扩展的错误码体系。

| 特性 | 说明 |
|------|------|
| 类型安全 | `Result<T, E>` 强制调用方显式处理错误 |
| 零开销 | 小对象优化 (SBO)、placement new 实现、编译期查表 |
| 错误级联 | 支持 `wrap()` 链式包装，保留完整错误上下文 |
| 源码追踪 | 自动记录错误发生的文件、行号、函数名 |
| 便捷宏 | `EF_TRY`、`EF_RETURN_IF_ERROR` 等宏简化错误传播 |

### 2. [日志模块](./log/) — `log`

高性能、线程安全的日志管理组件，支持多级日志过滤、批量缓冲写入、自动文件轮转。

| 特性 | 说明 |
|------|------|
| 五级日志 | `DEBUG` / `INFO` / `WARNING` / `ERROR` / `FATAL` |
| 高性能 | `snprintf` + 4KB 栈缓冲区，零堆分配 |
| 批量缓冲 | 减少系统调用，提升吞吐量 |
| 自动轮转 | 按行数切割，防止日志文件过大 |
| 双输出 | 控制台 / 文件可独立配置 |

### 3. [INI 解析器](./ini/) — `ini`

轻量级 C++17 INI 配置文件解析库，单例访问模式，支持隐式类型转换。

| 特性 | 说明 |
|------|------|
| 标准格式 | 支持 `[section]`、`key=value`、`#`/`;` 注释 |
| 隐式转换 | `Value` 自动转换为 `bool` / `int` / `double` / `string` |
| 全局键 | 未声明节的键值对自动归入 `Global` 节 |
| 零依赖 | 纯标准库实现，无需第三方库 |

### 4. [数据库连接池](./database-connection-pool/) — `database-connection-pool`

高性能、可扩展的数据库连接池，支持多种数据库类型，提供线程安全的连接管理和智能生命周期控制。

| 特性 | 说明 |
|------|------|
| 多数据库 | 抽象统一接口，当前已实现 MySQL 支持 |
| RAII 管理 | `PooledConnGuard` 自动归还连接，防止泄漏 |
| 自动扩容 | 根据实际需求动态调整连接数量 |
| 预处理语句 | 支持参数化查询，防止 SQL 注入 |
| 事务控制 | 完整的 `begin` / `commit` / `rollback` 接口 |

### 5. [Redis 连接池](./redis-connection-pool/) — `redis-connection-pool`

高性能、线程安全的 Redis 连接池，基于 `redis-plus-plus` 封装，提供自动化连接生命周期管理。

| 特性 | 说明 |
|------|------|
| RAII 守卫 | `RedisConnGuard` 自动归还连接，杜绝泄漏 |
| 自动重连 | 归还时检测有效性，无效连接自动销毁并重建 |
| 空闲回收 | 后台线程定期回收超时空闲连接 |
| 健康检查 | 基于 `PING` 命令的连接有效性检测 |
| 可观测性 | 内置统计信息，便于监控和调优 |

## 构建与集成

每个组件都是独立的，可单独编译。以 `log` 组件为例：

```bash
# 进入组件目录
cd log

# 创建构建目录
mkdir build && cd build

# 配置与编译
cmake ..
make
```

也可以将组件源码直接集成到你的项目中：

```cmake
# CMakeLists.txt 示例
add_subdirectory(path/to/backend-common-components/log)
target_link_libraries(your_target log)
```

或直接将 `include/` 加入头文件搜索路径，将 `src/` 下的源文件加入编译。
